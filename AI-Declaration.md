## รายงานฉบับนี้ นำปัญญาประดิษฐ์มาใช้ในขั้นตอนต่อไปนี้
## Sprint1
-   1)	ศึกษาคำสั่งใน robot นอกเหนือจากที่ศึกษา โดยมีการศึกษาการใช้ Test Teardown, Suite Teardown นอกเหนือจากการ setup library
-   2)	ศึกษาการกำหนด variables แบบที่ไม่กำหนดค่าชัดเจนแต่เป็นการกรอก input ใน terminal การใช้คำสั่งรัน
-   3)	ศึกษาการใช้ตัวอย่างคำสั่ง keywords แบบกำหนดการทำงานเป็นเซ็ตคำสั่ง และการกำหนดการใช้ log ขั้นตอนลง console เพื่อสะดวกต่อการอ่าน flow การทำงานรวมถึงการใช้ Arguments เก็บค่าที่ได้รับจาก keywords ที่เรียกใช้งาน
-   4)	ศึกษา library cron ที่ใช้กำหนดให้โปรแกรมทำการลบบัญชีผู้ใช้ตามช่วงเวลาที่ได้ตั้งไว้
-   5)	ใช้ในการตรวจเช็คแก้ไขปัญหา token ไม่ถูกส่งใน handleDelete() ซึ่งได้รับการตรวจเช็คและแก้ไขเรียบร้อยแล้ว
-   6)	ศึกษาการจัดการข้อผิดพลาด (Error Handling) การกำหนด Response ตามมาตรฐาน
-   7)	ช่วยสร้างไฟล์ทดสอบด้วย Robot Framework จากกรณีทดสอบที่ได้ออกแบบไว้
## Sprint2
-   1) ศึกษาและสรุป requirement ของฟังก์ชันสัมภาระเกินปกติ โดยช่วยจัด flow การทํางานระหว่างฝั่งผู้สร้างเส้นทาง (Driver) และผู้จอง (Passenger) ว่าควรแสดง/ไม่แสดงตัวเลือกเมื่อใด
-   2) ช่วยกําหนดและตรวจสอบโครงสร้างข้อมูล (Data Model) เช่นการเพิ่มฟิลด์ที่เกี่ยวข้องกับสัมภาระเกินใน Route/Booking และการกําหนดประเภทสัมภาระ (เช่น MEDIUM, LARGE,
EXTRA_LARGE)
-   3) ช่วยออกแบบกฎ Validation ฝั่ง Backend เช่นกรณีเลือกสัมภาระเกินแล้วต้องระบุประเภท และกรณีไม่เลือกแล้วห้ามส่งประเภท เพื่อป้องกันข้อมูลไม่ถูกต้อง
-   4) ช่วยวาง logic การคํานวณค่าบริการเพิ่มจากสัมภาระ โดยตรวจสอบให้สอดคล้องกับสิทธิ์ของ
route และบันทึกผลลงข้อมูลการจอง
-   5) ช่วยปรับปรุง Test Case อัตโนมัติด้วย Robot Framework ทดสอบในไฟล์
ExtraLuggage.robot ครอบคลุมกรณีผ่าน/ไม่ผ่าน/ข้าม (PASS/FAIL/SKIP) ทั้งฝั่ง
Passenger และ Driver
-   6) ช่วยวางแนวทาง API Test บน Postman
-   7) ช่วยวิเคราะห์ข้อผิดพลาดระหว่างทดสอบและแก้ไขเชิงเทคนิค เช่นเคส route ไม่พร้อมจอง, click
intercepted, และการ map ผลทดสอบให้ตรงกับพฤติกรรมระบบจริง
-   8) ช่วยปรับปรุง Test Case อัตโนมัติด้วย pytest ในกรณีของ Chat
-   9) ช่วยออกแบบหน้าต่าง UI ของ Chat ให้เป็น Widget
-   10) ช่วยแก้ไขการย่อและขยายหน้าต่างแชท
-   11) ช่วยแก้ไขปัญหาการแจ้งเตือนของแชทกรณีที่มีการส่งข้อความมาแต่สถานะแจ้งเตือนไม่ถูกต้อง และเพิ่มฟังก์ชัน MarkasRead()
-   12) ช่วยแก้ไขปัญหา Test Case ใน Robot Framework เพื่อทดสอบฟังก์ชัน Chat และช่วยเตรียม Data สำหรับเทสลบข้อมูลใน cleanup.robot
-   13) ช่วยปรับปรุง Test Case ในส่วนฟังก์ชัน Nearby Alert
-   14) ช่วยออกแบบหน้าต่างของ Driver Hub และเพิ่มเสียงเมื่อเวลามีการแจ้งเตือน
-   15) ช่วยออกแบบการแสดงผลการตรวจจับคำที่เข้าข่ายผิดกฎต่อหลัก Community Guideline&Policy
